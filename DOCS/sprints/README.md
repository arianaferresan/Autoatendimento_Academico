## Como preencher cada sprint

Cada pasta de sprint deve concentrar:

- objetivo do ciclo;
- período;
- backlog comprometido;
- evidências das entregas;
- riscos, impedimentos e aprendizados;
- registro explícito de `carry-over`, quando houver;
- vínculo com os itens do backlog do produto.

## Observação

O backlog do produto permanece em `DOCS/produto/`, enquanto cada sprint registra apenas o recorte comprometido naquele ciclo. Um item iniciado em uma sprint não deve ser marcado como concluído se o DoD do produto ainda não tiver sido atendido.
