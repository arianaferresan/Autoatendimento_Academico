## Como preencher cada sprint

Cada pasta de sprint deve concentrar:

- objetivo do ciclo;
- periodo;
- backlog comprometido;
- evidencias das entregas;
- riscos, impedimentos e aprendizados;
- registro explicito de `carry-over`, quando houver;
- vinculo com os itens do backlog do produto.

## Observacao

O backlog do produto permanece em `DOCS/produto/`, enquanto cada sprint registra apenas o recorte comprometido naquele ciclo. Um item iniciado em uma sprint nao deve ser marcado como concluido se o DoD do produto ainda nao tiver sido atendido.
